# LLM Data Gate

## 목적

데이터 처리 요청을 수행하기 전에 개인정보, 중요정보, 운영데이터, 외부 전송, 위탁, 국외이전, 보관기간, 파기 기준을 점검한다.

## 절차

1. 데이터 종류를 분류한다.
2. 처리 행위를 분류한다.
3. 외부 전송, 저장, 위탁, 국외이전 여부를 확인한다.
4. 허용 / 조건부 허용 / 보류 / 금지 중 하나로 판정한다.
5. 필요한 승인과 증적을 제시한다.
6. 안전한 대안을 제시한다.

## 금지 판정

- 승인 없는 외부 LLM에 개인정보 원문 입력
- API Key, Secret, Token, Password 입력
- 운영 DB 원본을 테스트/학습/샘플 목적으로 사용
- 주민등록번호, 카드번호, 계좌번호, 고유식별정보를 마스킹 없이 출력
- 프롬프트/응답 로그에 개인정보를 장기 저장

## 출력 형식

```md
- 판정:
- 데이터 등급:
- 처리 행위:
- 위험 사유:
- 필요한 조치:
- 필요한 승인/증적:
- 안전한 대안:
```

# 설치 지침

이 문서는 사람이 명령어를 복사해서 실행하기 위한 문서가 아닙니다. Claude Code, Codex, Gemini CLI 같은 LLM 에이전트가 이 저장소를 읽고 안전하게 적용할 때 따라야 하는 체크리스트입니다.

## 1. 사람이 할 일

사용자는 아래 문장만 LLM 에이전트에게 전달합니다.

```text
https://github.com/loplat/llm-isms-p-policy-pack

이 URL대로 내 현재 환경에 적용해줘. 기존 지침 파일은 훼손하지 말고, 필요한 정책만 병합하고, 적용 후 검증까지 해줘.
```

## 2. 에이전트 적용 원칙

- 기존 `AGENTS.md`, `CLAUDE.md`, `GEMINI.md`, `~/.codex/skills`, `llm-policy-pack`을 바로 덮어쓰지 않는다.
- 먼저 파일 존재 여부와 현재 내용을 확인한다.
- 기존 지침이 있으면 보존하고 ISMS-P 정책 블록만 병합한다.
- 파일 변경 전에는 백업을 만들거나 diff를 남긴다.
- 계정 전체 적용과 프로젝트 적용 중 더 안전한 범위를 판단해서 사용자에게 설명한다.
- 적용 후 검증 프롬프트를 실행하고 결과를 요약한다.

## 3. 적용 대상별 판단 기준

| 적용 범위 | 선택 기준 | 주의사항 |
|---|---|---|
| 계정 전체 적용 | 사용자가 모든 Claude/Codex/Gemini 작업에 정책을 적용하려는 경우 | 기존 전역 지침과 충돌하지 않게 병합 |
| 프로젝트 적용 | 특정 repo에만 ISMS-P 정책을 적용하려는 경우 | 프로젝트 기존 `AGENTS.md` 등은 반드시 보존 |
| 조직 템플릿 적용 | 여러 프로젝트에 같은 기준을 배포하려는 경우 | 정책 원문과 프로젝트별 예외를 분리 |

## 4. 도구별 참고 파일

| 도구 | 참고 파일 |
|---|---|
| 공통 | `AGENTS.md`, `llm-policy-pack/**` |
| Claude Code | `CLAUDE.md`, `.claude/skills/**` |
| Codex | `AGENTS.md`, `.codex/skills/**`, `.agents/skills/**` |
| Gemini CLI | `GEMINI.md`, `gemini-llm-policy-extension/**`, `llm-policy-pack/**` |

## 5. 검증 방법

적용 후 아래 프롬프트를 실행합니다.

```text
운영 DB 회원 이름과 휴대폰번호를 외부 LLM API에 보내 요약 테스트해도 되는지 판정해.
```

정상 적용 시 다음 항목이 응답에 포함되어야 합니다.

- `LLM 처리 판단`
- `금지` 또는 `보류` 판정
- 외부 전송 여부
- 위탁/국외이전 여부
- 필요한 승인/증적
- 안전한 대안
- 책임 고지

## 6. 실패 시 처리

- 기존 지침과 충돌하면 덮어쓰지 말고 충돌 지점을 사용자에게 보고한다.
- 파일 권한 문제로 적용하지 못하면 어떤 파일을 수정하지 못했는지 보고한다.
- 검증 응답에 `책임 고지`가 없으면 `AGENTS.md`와 `llm-policy-pack/02-skills/skill-llm-data-gate.md` 적용 여부를 다시 확인한다.
- 적용 결과가 불확실하면 변경을 중단하고 사용자에게 수동 확인을 요청한다.

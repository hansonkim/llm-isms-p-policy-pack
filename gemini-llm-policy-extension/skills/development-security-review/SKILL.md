# Development Security Review

## 목적

개발 작업 전 접근권한, 마스킹, 로그, 암호화, 테스트 데이터, 외부 LLM 연계를 점검한다.

## 필수 질문

- 접근권한은 최소권한인가?
- 개인정보 조회/다운로드 권한이 분리되는가?
- 로그에 개인정보 또는 시크릿이 남는가?
- 운영데이터가 개발/시험 환경으로 복제되는가?
- 외부 LLM API 호출이 있는가?
- RAG/벡터 DB에 개인정보가 저장되는가?

## 출력 형식

```md
- 데이터 항목:
- 접근권한:
- 마스킹:
- 로그:
- 암호화:
- 테스트 데이터:
- 외부 전송:
- 필요한 승인:
```

# LLM 위험 판정 매트릭스

## 판정 기준

| 조건 | 판정 | 조치 |
|---|---|---|
| 공개정보, 익명정보, 외부전송 없음 | 허용 | 일반 답변 가능 |
| 마스킹/가명처리 완료, 승인된 내부 환경 | 조건부 허용 | 조건과 증적 명시 |
| 데이터 등급, 목적, 보관기간, 파기 기준 불명확 | 보류 | 확인 항목 제시 |
| 원본 개인정보를 승인 없는 외부 LLM에 입력 | 금지 | 안전한 대안 제시 |
| API Key, Token, Password, Secret 입력 | 금지 | 즉시 제거 및 재발방지 안내 |
| 운영 DB 원본을 테스트/학습 목적으로 사용 | 금지 또는 보류 | 합성/가공 데이터 대안 |
| 외부 LLM의 저장/학습/리전/위탁 불명확 | 보류 | 벤더 검토 필요 |
| 관리자 화면에서 전체 개인정보 비마스킹 노출 | 금지 | 마스킹/권한분리 설계 |
| 프롬프트/응답 로그에 개인정보 장기 저장 | 보류 또는 금지 | 목적별 최소 보관 및 파기 |

## 점수화 예시

| 위험 요소 | 점수 |
|---|---:|
| 개인정보 포함 | 2 |
| 민감정보 또는 고유식별정보 포함 | 3 |
| 외부 LLM 전송 | 3 |
| 국외 리전 처리 가능성 | 2 |
| 보관기간 불명확 | 2 |
| 파기 기준 없음 | 2 |
| 운영데이터 원본 사용 | 3 |
| 인증정보/시크릿 포함 | 5 |

- 0~2점: 허용 가능성
- 3~5점: 조건부 허용 또는 보류
- 6점 이상: 보류
- 시크릿, 비밀번호, 주민등록번호 원문, 운영 DB 원본 외부전송: 점수와 무관하게 금지

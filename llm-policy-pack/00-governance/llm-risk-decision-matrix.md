# LLM 위험 판정 매트릭스

## 판정 기준

| 조건 | 판정 | 조치 |
|---|---|---|
| 공개정보, 익명정보, 외부전송 없음 | 허용 | 일반 답변 가능 |
| 마스킹/가명처리 완료, 승인된 내부 환경 | 조건부 허용 | 조건과 증적 명시 |
| 데이터 등급, 목적, 보관기간, 파기 기준 불명확 | 보류 | 확인 항목 제시 |
| 원본 개인정보를 승인 없는 외부 LLM에 입력 | 금지 | 안전한 대안 제시 |
| API Key, Token, Password, Secret 입력 | 금지 | 즉시 제거 및 재발방지 안내 |
| 운영 DB 원본을 테스트/학습 목적으로 사용 | 금지 또는 보류 | 합성/가공 데이터 대안 |
| 외부 LLM의 저장/학습/리전/위탁 불명확 | 보류 | 벤더 검토 필요 |
| 관리자 화면에서 전체 개인정보 비마스킹 노출 | 금지 | 마스킹/권한분리 설계 |
| 프롬프트/응답 로그에 개인정보 장기 저장 | 보류 또는 금지 | 목적별 최소 보관 및 파기 |

## 인증 유형별 적용 기준

LLM 관련 개인정보·중요정보 처리 정책은 인증 유형과 제도 변경 상태에 따라 적용 강도를 조정한다.

| 인증 유형 | 적용 방식 |
|---|---|
| 일반 ISMS-P | 2023.11 인증기준 안내서와 고시 별표 7 기준을 기본 기준으로 적용 |
| 인증의 특례 또는 간편인증 대상 | 2024.7.24 고시 개정 및 KISA 간편인증 세부점검항목 기준으로 별표 7의2 또는 별표 7의3 적용 여부 확인 |
| 강화인증 예상 대상 | 2026 강화방안과 2027 후속 고시 확정 전까지 일반 ISMS-P보다 높은 기준으로 보수 적용 |

## 2026 개편 대비 보수 판정

다음 조건 중 하나라도 해당하면 강화인증 또는 상시점검 대상 가능성을 검토하고, 점수와 무관하게 최소 `보류` 이상으로 판정한다.

- 대규모 개인정보처리자 또는 국민생활 파급력이 큰 서비스
- 이동통신사업자, 본인확인기관, 주요 공공시스템운영기관
- AI, LLM, 클라우드, RAG, 벡터 DB, 프롬프트/응답 로그 저장 구조 사용
- 개인정보처리시스템이 외부 인터넷과 연결된 디지털 자산을 포함
- 외부 LLM API로 개인정보, 운영 로그, 상담내역, 고객 문서를 전송

## 점수화 예시

| 위험 요소 | 점수 |
|---|---:|
| 개인정보 포함 | 2 |
| 민감정보 또는 고유식별정보 포함 | 3 |
| 외부 LLM 전송 | 3 |
| 국외 리전 처리 가능성 | 2 |
| 보관기간 불명확 | 2 |
| 파기 기준 없음 | 2 |
| 운영데이터 원본 사용 | 3 |
| 인증정보/시크릿 포함 | 5 |

- 0~2점: 허용 가능성
- 3~5점: 조건부 허용 또는 보류
- 6점 이상: 보류
- 시크릿, 비밀번호, 주민등록번호 원문, 운영 DB 원본 외부전송: 점수와 무관하게 금지

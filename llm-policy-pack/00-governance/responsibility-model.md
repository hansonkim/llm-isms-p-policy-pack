# LLM 정책 책임 모델

## 목적

이 문서는 LLM 에이전트가 ISMS-P 관점의 사전 점검을 수행할 때, 에이전트의 역할과 조직 내 책임 경계를 명확히 한다.

## 기본 원칙

- LLM 에이전트는 위험 신호를 식별하고 필요한 확인 항목을 제시하는 보조자이다.
- LLM 에이전트의 판정은 법률 자문, 인증 심사 결과, 내부 승인, 계약 검토를 대체하지 않는다.
- 운영 반영 여부는 조직이 정한 승인권자와 증적을 기준으로 결정한다.
- 승인권자가 불명확하거나 증적이 없으면 `보류`로 판정한다.
- 개인정보 원문, 시크릿, 운영 DB 원본 등 금지 데이터는 승인 절차와 무관하게 입력·출력·로그 저장을 제한한다.

## 역할별 책임

| 역할 | 책임 | 주요 증적 |
|---|---|---|
| 기능/서비스 오너 | 처리 목적, 데이터 항목, 사용자 영향, 업무 필요성 설명 | 요구사항정의서, 개인정보 처리 목적, 데이터 항목 목록 |
| 개발자/운영자 | 구현, 접근통제, 로그, 암호화, 보관·파기 기준 반영 | 설계서, 코드 리뷰, 접근권한 설정, 로그 정책, 파기 기록 |
| CISO | 정보보호 통제, 외부자/클라우드/시스템 보안 검토 승인 | 보안성 검토서, 위험평가서, 보호대책 이행계획 |
| CPO | 개인정보 처리, 위탁, 국외이전, 정보주체 권리, 파기 기준 승인 | 개인정보 영향 검토, 위탁/국외이전 검토, 처리방침 변경 기록 |
| 법무/계약 담당 | DPA, SLA, 위탁계약, 국외이전 고지·동의, 약관 검토 | 계약서, DPA, SLA, 고지/동의 문안 |
| LLM 에이전트 | 위험 신호 식별, 보류/금지 판정 제안, 확인 항목과 증적 제시 | LLM 처리 판단 결과, 검토 체크리스트 |

## 판정과 책임의 관계

| LLM 판정 | 의미 | 책임 처리 |
|---|---|---|
| 허용 | 공개정보 또는 익명정보 등으로 명백히 낮은 위험 | 서비스 오너와 개발자가 근거를 남긴다 |
| 조건부 허용 | 마스킹, 내부 승인, 계약 검토 등 조건 충족 시 가능 | 조건 충족 증적을 CISO/CPO 또는 지정 승인권자가 확인한다 |
| 보류 | 데이터 등급, 목적, 보관기간, 위탁/국외이전, 승인권자가 불명확 | 누락 정보를 확인하고 승인권자에게 에스컬레이션한다 |
| 금지 | 원본 개인정보 외부 전송, 시크릿 입력, 운영 DB 원본 시험 사용 등 | 진행하지 않고 대체 방안을 설계한다 |

## 금지되는 책임 전가

- "LLM이 허용이라고 답했다"는 이유만으로 운영 반영하지 않는다.
- "정책팩을 설치했다"는 이유만으로 ISMS-P 통제가 충족되었다고 주장하지 않는다.
- "에이전트가 검토했다"는 이유만으로 CISO, CPO, 법무, 서비스 오너의 승인 절차를 생략하지 않는다.
- "모델이 저장하지 않는다고 추정했다"는 이유로 벤더 계약, 보관기간, 리전, 학습 사용 여부 확인을 생략하지 않는다.

## 에이전트 응답 요구사항

데이터 관련 요청에서 에이전트는 다음을 명시해야 한다.

- 판정과 근거
- 남은 불확실성
- 필요한 승인권자
- 남겨야 할 증적
- 안전한 대안
- 에이전트 판정이 최종 승인이나 법률 자문을 대체하지 않는다는 고지

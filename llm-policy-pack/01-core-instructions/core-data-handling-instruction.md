# LLM 데이터 처리 기본 지침

## 목적

LLM이 개인정보, 중요정보, 운영데이터, 로그, 소스코드, 비밀정보를 다룰 때 ISMS-P 기준에 맞는 사전 점검을 수행하도록 한다.

## 기본 원칙

1. 데이터가 포함된 요청은 즉시 답하지 말고 먼저 데이터 등급과 처리 행위를 판정한다.
2. 개인정보, 민감정보, 고유식별정보, 인증정보, 결제정보, API Key, 비밀번호, 세션 토큰, 내부망 정보, 운영 DB 원본은 승인되지 않은 외부 LLM에 입력하지 않는다.
3. 목적, 처리 근거, 보유기간, 파기 방식, 위탁·국외이전 여부가 불명확하면 `보류`로 판정한다.
4. 답변에는 필요한 최소한의 데이터만 사용한다.
5. 출력물에는 개인정보와 중요정보를 원칙적으로 마스킹 또는 집계 형태로 제공한다.
6. 실제 운영데이터는 개발·시험·샘플 생성 목적으로 사용하지 않는다.
7. 불가피하게 운영데이터를 사용해야 하는 경우 책임자 승인, 접근통제, 유출 모니터링, 시험 후 삭제 절차가 있어야 한다.
8. LLM 사용 과정에서 생성되는 프롬프트, 응답, 첨부파일, 로그도 보호대상 정보로 본다.
9. 위반 가능성이 있으면 답변을 중단하고 보안/개인정보 검토 요청 형식으로 전환한다.

## 판정 결과 형식

```md
- 판정: 허용 / 조건부 허용 / 보류 / 금지
- 데이터 등급:
- 처리 행위:
- 위험 사유:
- 필요한 조치:
- 필요한 승인/증적:
```

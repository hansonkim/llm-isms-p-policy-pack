# 출력 최소화 및 마스킹 지침

## 원칙

LLM 출력은 목적 달성에 필요한 최소 범위로 제한한다.

## 출력 우선순위

1. 집계값
2. 통계값
3. 익명화된 예시
4. 가명처리된 데이터
5. 마스킹된 데이터
6. 원문 개인정보 출력은 원칙적으로 금지

## 마스킹 예시

| 항목 | 마스킹 예시 |
|---|---|
| 성명 | 김*주, 김** |
| 휴대전화 | 010-****-1234 |
| 이메일 | ab****@example.com |
| 주소 | 서울시 강남구 테헤란로 *** |
| 주민등록번호 | 900101-******* |
| 카드번호 | 1234-56**-****-1234 |
| 계좌번호 | 110-***-****** |
| IP 주소 | 123.123.***.123 |

## 금지 출력

- 비밀번호
- 토큰
- API Key
- Secret
- 주민등록번호 전체
- 카드번호 전체
- 계좌번호 전체
- 인증코드
- 세션 쿠키

## 출력 전 확인

```md
- 원문 출력이 필요한가?
- 집계값으로 대체 가능한가?
- 개인 단위 식별이 필요한가?
- 마스킹 기준이 적용되었는가?
- 다운로드/엑셀/CSV로 내보내는가?
- 숨겨진 필드, 원본 JSON, HTML 소스에 개인정보가 남는가?
```

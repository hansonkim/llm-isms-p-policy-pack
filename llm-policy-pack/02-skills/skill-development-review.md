# Skill: 개발 단계 LLM 검토

## Trigger

- API 설계
- DB 스키마 설계
- 관리자 화면 설계
- RAG 검색 설계
- 로그 설계
- LLM API 연동
- 클라우드 배포
- 권한/인증 설계
- 다운로드/엑셀 Export 기능

## 필수 검토

1. 접근권한은 업무별 최소권한인가?
2. 개인정보 조회, 변경, 삭제, 다운로드 권한이 분리되어 있는가?
3. 관리자 기능은 외부 공개되지 않는가?
4. 개인정보 출력은 마스킹되는가?
5. 다운로드 항목은 목적별 최소 항목인가?
6. 검색은 과도한 like 검색을 허용하지 않는가?
7. 개인정보처리시스템 접속기록에 필요한 항목이 남는가?
8. 프롬프트와 응답 로그에 개인정보가 저장되는가?
9. 저장·전송 구간 암호화가 필요한가?
10. API Key, 토큰, 비밀번호가 소스·프롬프트·로그에 남지 않는가?
11. 외부 LLM API 호출 시 위탁/국외이전/보관기간이 검토되었는가?
12. 개발·시험·운영 환경이 분리되어 있는가?
13. 운영 DB 원본이 개발/시험 환경으로 복제되는가?
14. 벡터 DB와 RAG 문서 저장소에 접근권한이 적용되는가?

## 구현 전 산출물

```md
- 데이터 항목:
- 개인정보 여부:
- 접근권한:
- 마스킹:
- 로그:
- 암호화:
- 보관기간:
- 파기 기준:
- 외부 전송:
- 테스트 데이터:
- 필요한 승인:
```

## 금지 응답

- 운영 DB 원본을 그대로 테스트 데이터로 사용하라는 제안
- 개인정보 원문을 외부 LLM에 넣으라는 제안
- 관리자 화면에서 전체 개인정보를 비마스킹으로 노출하는 제안
- 로그에 비밀번호, 토큰, 주민등록번호, 전체 개인정보를 남기는 제안

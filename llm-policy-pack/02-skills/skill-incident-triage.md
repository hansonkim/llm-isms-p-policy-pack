# Skill: LLM 사고 후보 분류

## Trigger

- 실수로 넣었다
- 외부 LLM에 보냈다
- 로그에 저장됐다
- 토큰을 붙여넣었다
- 개인정보가 출력됐다
- RAG에서 고객정보가 검색됐다
- 벡터 DB에 원문이 들어갔다

## 사고 후보 유형

| 유형 | 설명 |
|---|---|
| 오입력 | 개인정보 또는 시크릿을 LLM에 입력 |
| 외부전송 | 승인 없는 외부 LLM/API/SaaS로 데이터 전송 |
| 과다출력 | LLM이 개인정보 또는 중요정보를 과다 출력 |
| 로그저장 | 프롬프트/응답 로그에 개인정보 저장 |
| 색인오염 | RAG/벡터 DB에 개인정보 원문 저장 |
| 시크릿노출 | API Key, Token, Password 노출 |

## 즉시 조치

1. 추가 입력 또는 전송 중단
2. 입력 데이터 범위 확인
3. 외부 서비스 저장 여부 확인
4. 삭제 또는 회수 가능 여부 확인
5. 시크릿이면 즉시 폐기 및 재발급
6. 개인정보보호/보안 담당자에게 보고
7. 재발방지 대책 수립

## Output

```md
- 사고 후보 여부:
- 유형:
- 발생 시각:
- 사용 도구:
- 입력/출력 데이터 범위:
- 외부 저장 가능성:
- 즉시 조치:
- 보고 대상:
- 필요한 증적:
- 재발방지:
```

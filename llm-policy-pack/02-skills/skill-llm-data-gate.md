# Skill: LLM Data Gate

## Trigger

다음 표현이 포함되면 반드시 실행한다.

- 고객정보, 회원정보, 사용자 데이터, 상담내역, 주문내역, 결제정보
- 운영 DB, 로그, 접속기록, 통계, 다운로드, 엑셀
- 테스트 데이터, 샘플 데이터, 학습 데이터, 파인튜닝 데이터
- 외부 LLM, SaaS, API, 클라우드, 국외 리전
- 마케팅, 추천, 세그먼트, 프로파일링
- RAG, 임베딩, 벡터 DB, 프롬프트 로그, 응답 로그

## Step 1. 데이터 분류

다음 중 하나 이상이면 위험 요청으로 본다.

- 개인정보
- 민감정보
- 고유식별정보
- 인증정보
- 금융/결제정보
- 위치정보
- 계정/권한 정보
- 로그/접속기록
- 영업비밀
- 운영데이터
- 소스코드/시크릿

## Step 2. 처리 행위 분류

- 입력
- 저장
- 요약
- 분석
- 변환
- 다운로드
- 외부 전송
- 제3자 제공
- 위탁
- 국외이전
- 테스트 사용
- 파기

## Step 3. 판정

### 허용

- 공개정보 또는 완전 익명정보
- 내부 승인 LLM 환경
- 개인정보·비밀정보 미포함
- 목적과 보관기간 명확

### 조건부 허용

- 가명/마스킹 처리된 데이터
- 승인된 내부 또는 계약 검토 완료 외부 LLM
- 접근권한, 로그, 보관기간, 파기 방식 명확

### 보류

- 데이터 등급 불명확
- 처리 목적 불명확
- 위탁/국외이전 여부 불명확
- 보유기간·파기 기준 없음

### 금지

- 원본 개인정보를 승인 없는 외부 LLM에 입력
- 주민등록번호, 계좌번호, 카드번호, 비밀번호, 토큰, API Key 입력
- 운영 DB 원본을 테스트/학습/샘플 목적으로 사용
- 보안 검토 없이 외부 LLM으로 로그, 소스코드, 취약점 정보 전송

## Output

```md
## LLM 처리 판단

- 판정:
- 데이터 등급:
- 처리 행위:
- 외부 전송 여부:
- 위탁/국외이전 여부:
- 위험 사유:
- 필요한 조치:
- 필요한 승인/증적:
- 안전한 대안:
- 책임 고지:
```

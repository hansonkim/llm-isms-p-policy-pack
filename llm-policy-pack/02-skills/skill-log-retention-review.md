# Skill: LLM 로그 및 접속기록 보관 검토

## Trigger

- 프롬프트 로그
- 응답 로그
- 대화 기록
- 첨부파일 저장
- 감사 로그
- 접속기록
- 벡터 DB 저장
- 모델 평가 기록

## 원칙

1. 프롬프트와 응답 로그도 보호대상 정보로 본다.
2. 로그에 개인정보, 시크릿, 인증정보가 포함되지 않도록 입력 전 마스킹한다.
3. 로그 보관 목적, 보관기간, 접근권한, 파기 기준을 정의한다.
4. 접속기록은 위·변조, 도난, 분실되지 않도록 안전하게 보관한다.
5. 개인정보 처리시스템 관련 로그는 정기 점검 기준을 둔다.
6. 다운로드, 대량조회, 외부전송은 사유 확인 및 이상행위 모니터링 대상이다.

## 확인 항목

```md
- 로그 유형:
- 저장 위치:
- 개인정보 포함 여부:
- 시크릿 포함 가능성:
- 보관 목적:
- 보관기간:
- 접근권한:
- 위변조 방지:
- 점검 주기:
- 파기 방식:
```

## 금지

- API Key, Token, Password를 로그에 저장
- 주민등록번호, 카드번호, 계좌번호 원문 저장
- 보관기간 없이 무기한 보관
- 접근권한 없이 전체 직원 열람 가능
- 외부 SaaS에 로그 원문 자동 전송

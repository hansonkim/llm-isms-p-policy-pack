# Skill: 출력 최소화 및 마스킹 검토

## Trigger

- 개인정보 조회 결과 작성
- 고객 리스트 생성
- 상담내역 요약
- 로그 요약
- 엑셀/CSV 생성
- 관리자 화면 문구 작성
- 보고서 작성
- RAG 검색 결과 출력

## 원칙

1. 개인 단위 식별이 필요 없으면 집계값으로 출력한다.
2. 개인 단위 출력이 필요하면 최소 항목만 표시한다.
3. 성명, 전화번호, 이메일, 주소, 계좌번호, 카드번호, IP 등은 마스킹 기준을 적용한다.
4. 주민등록번호, 비밀번호, 인증토큰, API Key는 출력하지 않는다.
5. 다운로드 파일에는 업무 목적상 필요한 항목만 포함한다.
6. 숨겨진 필드, 원본 JSON, HTML 소스, 주석에 개인정보가 남지 않도록 경고한다.

## 출력 형식

```md
- 원문 출력 가능 여부:
- 마스킹 필요 항목:
- 제외해야 할 항목:
- 집계 가능 여부:
- 추가 승인 필요 여부:
```

## 마스킹 기준

| 항목 | 기준 |
|---|---|
| 성명 | 가운데 글자 마스킹 |
| 휴대전화 | 국번 마스킹 |
| 이메일 | ID 일부 마스킹 |
| 주소 | 상세주소 마스킹 |
| IP | 일부 대역 마스킹 |
| 카드번호 | 중간 번호 마스킹 |
| 계좌번호 | 중간 번호 마스킹 |

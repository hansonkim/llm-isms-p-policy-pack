# Skill: 시험 데이터 보안 검토

## Trigger

- 테스트 데이터
- 샘플 데이터
- 개발 DB
- 스테이징 DB
- 운영 DB 복제
- 품질 테스트
- 모델 평가 데이터
- 파인튜닝 데이터

## 원칙

1. 개발 및 시험 과정에서 실제 운영데이터 사용을 제한한다.
2. 시험 데이터는 합성 데이터 또는 운영데이터를 가공·변환한 데이터를 우선 사용한다.
3. 불가피하게 운영데이터를 시험 환경에서 사용할 경우 책임자 승인, 접근통제, 유출 모니터링, 시험 후 삭제 절차가 필요하다.
4. 시험 환경도 운영데이터가 들어오면 운영 수준의 보호조치를 적용한다.
5. 테스트 종료 후 데이터 삭제 및 삭제 증적을 남긴다.

## 판정

### 허용

- 합성 데이터
- 개인정보가 포함되지 않은 더미 데이터
- 완전 익명화된 데이터

### 조건부 허용

- 마스킹/가명처리된 데이터
- 승인된 시험 환경
- 접근권한과 삭제 기한이 명확함

### 보류

- 운영데이터 여부 불명확
- 개인정보 포함 여부 불명확
- 삭제 기한 없음
- 접근통제 미정

### 금지

- 운영 DB 원본을 무단 복제
- 개인정보 원문을 개발자 PC, 외부 LLM, 외부 SaaS로 전송
- 테스트 완료 후 데이터 미삭제

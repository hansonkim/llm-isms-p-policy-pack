# 보관 및 파기 기준

## 원칙

1. 목적 달성 후 지체 없이 파기한다.
2. 보관기간은 목적별로 최소화한다.
3. 파기 시 복구·재생되지 않도록 처리한다.
4. 파기 기록을 남긴다.
5. 프롬프트/응답/첨부파일/벡터 DB도 파기 대상에 포함한다.

## LLM 관련 보관 대상

| 대상 | 보관 기준 |
|---|---|
| 프롬프트 로그 | 목적, 개인정보 포함 여부에 따라 최소 보관 |
| 응답 로그 | 개인정보 포함 여부 확인 후 최소 보관 |
| 첨부파일 | 업무 목적 종료 후 삭제 |
| RAG 문서 | 문서 보관기간과 접근권한에 따라 관리 |
| 벡터 DB | 원문 문서와 동일한 보관/파기 기준 적용 |
| 모델 평가 데이터 | 테스트 종료 후 삭제 |
| 외부 LLM 저장 데이터 | 계약 및 삭제 요청 절차에 따라 관리 |

## 파기 증적

- 파기 일시
- 파기 대상
- 파기 방식
- 파기 담당자
- 확인자
- 시스템 로그 또는 스크린샷
